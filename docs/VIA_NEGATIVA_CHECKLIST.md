# VIA_NEGATIVA_CHECKLIST

This checklist is for `aoa-playbooks` as the scenario composition and
recurring operations.

## Keep intact

- playbooks with real scenario distinctions
- explicit fallback paths and re-entry gates
- references to owner surfaces instead of copied owner instructions

## Merge, move, suppress, quarantine, deprecate, or remove when found

- scenario clones distinguished mostly by example names
- stress lanes that add no distinct threshold or fallback meaning
- copied owner commands embedded in playbooks

## Questions before adding anything new

1. Is this a new scenario family or just another example?
2. Does this lane change thresholds, fallback, or evidence expectations?
3. Can the playbook point to an owner surface instead of restating it?

## Safe exceptions

- a new recurring operation with clearly distinct operator flow
- temporary migration playbooks with sunset

## Exit condition

- Playbooks should recur like good rituals, not multiply like mirrors.
