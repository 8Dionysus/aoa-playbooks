# Checkpoint distillation and closed-loop pilot

This note anchors the `checkpoint-distillation-closed-loop-pilot` playbook.
It stays scenario-owned and does not replace the playbook canon.

## Core rule

Checkpoint distillation is a bounded review and condensation move.
A closed-loop pilot runbook is a bounded pilot guide, not execution authority.

## What this route is

This route preserves one reviewed checkpoint, distills the next honest move,
and packages the result as a pilot runbook that stays reviewable.

## What this route is not

- It is not live runtime authority.
- It is not a release contract.
- It is not a replacement for owner-local review.
- It is not a hidden scheduler or pilot runner.

## Boundary

The playbook may organize the path from a checkpoint to a pilot runbook.
It does not own the meaning of the checkpoint, the proof posture of the pilot,
or any downstream execution canon.
