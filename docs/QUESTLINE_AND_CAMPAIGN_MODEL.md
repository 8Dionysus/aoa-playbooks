# Questline and Campaign Model

## Purpose

This note defines the first playbook-owned RPG reflection surfaces for `aoa-playbooks`.

It exists so that multi-stage quest work can be read as a reviewed outline rather than as a hidden runtime ledger.

## Core rule

A questline is a reviewed outline of linked source-owned quests.

A campaign is a reviewed grouping of questlines that share a larger objective.

A raid is a rare coordinated route that needs explicit multi-role or multi-tier choreography.

None of these surfaces are runtime state.

## Questline

Use a questline when:

- there is one bounded multi-stage lane
- stage order matters
- anchor and reanchor rules should stay explicit
- reviewed harvest posture matters

Do not use a questline for one isolated quest or for a vague aspiration.

## Campaign

Use a campaign when:

- several questlines share one larger horizon
- the lanes remain distinguishable
- evidence can still be reviewed lane by lane

Do not use campaign language to hide unbounded sprawl.

## Raid

Use a raid only when:

- several roles or tiers need explicit coordination
- failure costs are high enough to justify extra ceremony
- anchor classes and reanchor rules are named up front

Raid should remain rare. A difficult route is not automatically a raid.

## Reanchor

Reanchor remains governed return, not retry.

Questline or campaign reflection may name:
- valid anchor classes
- maximum reanchors
- stop conditions after anchor loss

## Harvest

Questline and campaign surfaces may state a harvest posture, but they do not become the harvest authority themselves.

Good first-wave posture:
- use playbooks to name outline and recurrence shape
- leave final promotion decisions reviewable
- avoid pretending that one observed lane is already canonized recurrence

## Upstream refs

Example outline surfaces may cite upstream read-only quest refs such as `AOA-SK-Q-0003`.
Those refs stay source-owned upstream and do not widen this rollout into `aoa-skills`.

## Anti-patterns

- turning playbooks into a run ledger
- turning every sequence into a campaign
- using raid language for ordinary multi-file work
- hiding missing anchors behind continuity language
